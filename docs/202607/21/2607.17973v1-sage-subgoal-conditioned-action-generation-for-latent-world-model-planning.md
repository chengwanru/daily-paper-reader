## 摘要
潜在世界模型通过学习动作条件预测动态，并将其作为内部模拟器来想象和评估候选动作序列，已成为一种强大的规划范式。然而，随着规划 horizons 的增长，性能越来越受到候选提案质量的限制：固定的候选预算必须在指数级增长的动作空间中进行搜索，使得世界模型难以接触到高质量的候选未来进行评估。本文提出了一种先验条件规划器，用结构化指导替代随机提案初始化。在每个规划阶段，一个目标条件生成器预测指定时长的下一个可达潜在子目标，然后以此为基础生成候选动作序列。为了捕捉跨时间尺度的语义信息，我们使用不同时长的子目标作为先验，平衡细粒度的局部控制与更高层次的长期进展。随后，冻结的世界模型在执行前评估并优化这些由子目标条件生成的提案。在 PushT 和 OGBench Cube 上的实验表明，将潜在子目标分解与先验条件动作生成相结合，显著提升了长期规划性能，同时保留了强大的短期性能。具体而言，当目标偏移量为 150 时，PushT 的成功率从 12.7% 提升至 64.7%，OGBench Cube 的成功率从 26.7% 提升至 67.3%。

## Abstract
Latent world models have emerged as a powerful planning paradigm by learning action-conditioned predictive dynamics and using them as internal simulators to imagine and evaluate candidate action sequences. However, as the planning horizon grows, performance becomes increasingly constrained by proposal quality: a fixed candidate budget must search an exponentially larger action space, making it difficult to expose the world model to high-quality candidate futures for evaluation. In this paper, we introduce a prior-conditioned planner that replaces random proposal initialization with structured guidance. At each planning stage, a goal-conditioned generator predicts the next reachable latent subgoal for a specified duration, which is then used to condition the generation of candidate action sequences. To capture semantic information across temporal scales, we use subgoals of varying durations as priors, balancing fine-grained local control with higher-level long-horizon progress. Then the frozen world model evaluates and refines these subgoal-conditioned proposals before execution. Experiments on PushT and OGBench Cube show that coupling latent subgoal decomposition with prior-conditioned action generation substantially improves long-horizon planning while preserving strong short-horizon performance. To be specific, when the target offset is $150$, it raises PushT success from $12.7\%$ to $64.7\%$ and OGBench Cube success from $26.7\%$ to $67.3\%$.

### 论文总结：SAGE：基于子目标条件动作生成的潜在世界模型规划

#### 1. 核心问题与研究动机
潜在世界模型是一种强大的规划范式，它通过学习动作对潜在状态演变的预测，作为内部模拟器来评估候选动作序列。然而，其性能在**面对长时域规划任务时严重受限**。根本原因在于，当规划时域（Horizon）延长时，在固定的候选预算下，从通用随机分布中采样的动作序列很难覆盖指数级增长的巨大动作空间，从而导致难以生成或接触到能达成遥远目标的高质量候选行动。因此，核心问题是：**如何在不修改或重训世界模型的前提下，为规划器提供结构化的、高质量的候选动作提案，以显著提升其长时域规划能力？**

#### 2. 方法论：SAGE 规划框架

**核心思想**：将长时域规划分解为两个紧密耦合的步骤：
1.  **预测局部可到达的子目标**：在每一步规划中，生成一个在近期（例如未来15或25步）可达的潜在子目标。
2.  **基于子目标生成动作**：将预测出的子目标作为结构化先验，条件化地生成一系列候选动作序列，而不是随机初始化。

**关键技术细节与流程**：

-   **前置条件**：使用一个冻结的（frozen）潜在世界模型（LeWM）作为固定的内部模拟器和动作评估器。
-   **模型组件一：目标条件子目标生成器 (Goal-Conditioned Subgoal Generator)**
    -   **输入**：当前观测历史、远程目标、当前低维状态、剩余目标偏移量 δ、请求的规划时长 τ。
    -   **架构**：一个四层 Transformer 解码器。它以远程目标的潜在状态为基准，通过残差学习的方式预测未来 τ 步后的潜在状态 `ẑ_{t+τ}`。这种做法使生成的子目标既接近全局目标，又符合局部可行性。
    -   **训练**：使用专家轨迹中的对齐窗口，以最小化预测子目标与专家轨迹中对应时刻的真实潜在状态之间的 SmoothL1 损失和余弦相似度损失。
-   **模型组件二：子目标条件动作生成器 (Subgoal-Conditioned Action Generator)**
    -   **输入**：历史、状态、生成的子目标 `ẑ_{t+τ}`、远程目标以及时间变量 (δ, τ)。
    -   **架构**：一个三层 Transformer 解码器，输出一个**轨迹级高斯混合模型（GMM）**。该模型直接生成完整的、长度为 τ 的动作序列作为候选。
    -   **训练**：训练损失是专家动作序列在生成器输出的 GMM 下的负对数似然，并使用了冻结的子目标生成器的输出，以匹配在线规划时的场景。
-   **在线规划流程**：
    1.  **分解**：将总时域 H 分解为一系列规划阶段，每个阶段有一个特定的规划时长 τ。
    2.  **生成子目标**：在当前阶段，子目标生成器预测未来 τ 步的可达子目标。
    3.  **生成动作**：动作生成器基于该子目标，采样 K 个长度为 τ 的候选动作序列（从 GMM 中采样）。
    4.  **评估与优化**：冻结的 LeWM 对这些候选序列进行预测和评分（评估其预测未来与子目标的一致性），然后使用交叉熵方法（CEM）对评分最高的“精英”候选序列进行迭代优化。
    5.  **执行**：执行优化后的动作序列，更新环境状态，并进入下一个规划阶段。

#### 3. 实验设计

-   **任务/场景**：两个标准机器人操作任务的基准测试集：
    -   **PushT**：一个推 T 形物体的平面接触操控任务。
    -   **OGBench Cube**：涉及手臂运动、抓取和放置物体的复合任务。
-   **基准测试**：使用冻结的 LeWM 作为基础世界模型，评估规划成功率。规划时域 (H) 从 25 步到 150 步不等。
-   **对比方法**：
    -   **LeWM (Baseline)**：使用标准的高斯分布随机初始化 CEM 提案。
    -   **PRISM**：使用基于状态条件的动作先验初始化 CEM。
    -   **Generator-only**：使用生成的子目标作为 LeWM 的评估目标，但用原始的随机动作提案。
    -   **SAGE prior top**：直接执行动作生成器输出的最高权重动作序列，不使用 LeWM 进行评分和CEM优化。
    -   **Full SAGE**：完整方法，包含生成的子目标、多模态的 GMM 动作提案和 LeWM 引导的精英 CEM 优化。

#### 7. 优点

-   **方法创新性**：巧妙地将“目标分解”和“动作提案”结合起来，将一个纯优化问题转化为一个有结构先验的搜索问题，为解决长时域规划瓶颈提供了一个优雅且有效的新思路。
-   **即插即用**：SAGE 作为一个**插件式模块**，可以无缝地增强任何已训练好的冻结世界模型，无需重新训练该世界模型，具有很强的实用性和兼容性。
-   **时间灵活性**：支持**可变规划时长 (τ)**，使得同一框架能够灵活地处理不同粒度的规划需求，兼顾了局部精细控制和全局进程推进。
